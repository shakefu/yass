round-05-scale oracle -- how to run
===================================

The oracle is PRIVATE. It is never copied into a candidate's workspace; it only
ever runs from this directory, against a candidate binary it invokes itself.

Files
-----
  oracle/grade.py   The authoritative grader. Embeds an independent simulator
                    (simulate_run / simulate_audit + dispatch) and the batch
                    table. Every batch carries a hand-pinned expected
                    (stdout, stderr, exit). Compares a candidate's stdout,
                    stderr, and exit code byte-for-byte (UTF-8 with
                    surrogateescape -- no whitespace or trailing-newline
                    normalization).
  oracle/ref.py     A clean reference implementation of the scale CLI. Coded
                    independently of grade.py's simulators (straight if-ladder
                    vs data-driven registry), so a shared bug cannot pass both.
                    It must score full marks.

The CLI under test
------------------
One binary `scale`; argv[1] selects a subcommand from the closed set
{run, audit}. The grader appends the subcommand and feeds each batch's stdin.

  scale run    < operations  -> one ticket line per success (TARE ack / NET /
                                W0x verdict), one error line per rejected record
                                to stderr, exit = max severity.
  scale audit  < tickets      -> five-line class tally to stdout, exit 0.
  (missing / unknown / wrong-case / bare `-` argv[1]) -> "usage: scale {run|audit}"
                                on stderr, exit 2, no stdout.

Self-check (no candidate needed)
--------------------------------
Confirms the oracle's own invariants: registry/precedence integrity, the
classify() first-match boundary truth table (W01/W02/W03/W04 strict guards and
the W03-beats-W04 deciding case), four-corner W04 isolation, and that the
embedded simulator reproduces every hand-pinned batch expectation byte-exactly.

    cd test-specs/round-05-scale
    python3 oracle/grade.py --self-check

Expect exactly:

    SELFTEST OK

Grade a candidate
-----------------
Pass the base command that runs the candidate program; the grader appends the
subcommand (run / audit) and feeds each batch's stdin.

    cd test-specs/round-05-scale
    python3 oracle/grade.py --cmd "<run command>"

Examples:

    python3 oracle/grade.py --cmd "python3 /path/to/candidate/scale.py"
    python3 oracle/grade.py --cmd "node /path/to/candidate/scale.js"
    python3 oracle/grade.py --cmd "/path/to/candidate/scale"

The grader prints any per-batch FAIL diagnostics (expected vs got, repr'd so
whitespace and control bytes are visible) and a final line:

    SCORE: <passed>/<total>

A full score means passed == total.

Verify the reference scores full
--------------------------------
    cd test-specs/round-05-scale
    python3 oracle/grade.py --cmd "python3 oracle/ref.py"

Expect SCORE: <total>/<total>.
